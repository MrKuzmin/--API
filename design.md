# 🚗 Garage (Парковка МКД) — Дизайн-документ

---

## 1. Цель проекта

Разработать систему управления парковкой многоквартирного дома.  
Система фиксирует выезды и въезды автомобилей, контролирует количество свободных мест, разграничивает доступ: жильцы — только просмотр, администратор — полное управление.

---

## 2. Основные сущности

### 2.1. `residents` — жильцы и администратор
| Поле | Тип | Описание |
|------|-----|----------|
| id | SERIAL | уникальный идентификатор |
| full_name | VARCHAR(100) | ФИО |
| apartment | VARCHAR(10) | номер квартиры |
| birth_date | DATE | дата рождения (для проверки возраста) |
| has_license | BOOLEAN | наличие водительских прав |
| phone | VARCHAR(20) | номер телефона (уникальный, формат `+7...`) |
| role | VARCHAR(20) | `resident` (по умолчанию) |

> Администратор **не хранится в БД**. Логин и пароль заданы в коде (`admin` / `admin`).

### 2.2. `cars` — автомобили
| Поле | Тип | Описание |
|------|-----|----------|
| id | SERIAL | уникальный идентификатор |

| brand | VARCHAR(50) | марка |
| model | VARCHAR(50) | модель |
| color | VARCHAR(30) | цвет |

---

### 2.3. `parking_log` — журнал парковки
| Поле | Тип | Описание |
|------|-----|----------|
| id | SERIAL | уникальный идентификатор |

| departure_time | TIMESTAMP | время выезда |
| return_time | TIMESTAMP | время возвращения (`NULL` пока не вернулась) |

## 3. Бизнес-правила

### 3.1. Общие
- Парковка имеет **100 мест**.
- У жильца может быть несколько автомобилей.
- У квартиры — без ограничений.
- Машиной может управлять любой совершеннолетний жилец с правами (`has_license = true`).
- Водитель и владелец могут быть разными людьми.
- Один водитель может управлять **только одной машиной одновременно** (нет активной поездки).

### 3.2. Выезд автомобиля (админ)
1. Машина существует и находится на парковке (`status = 'parked'`).
2. Водитель достиг 18 лет (`birth_date`).
3. Водитель имеет права (`has_license = true`).
4. Водитель не находится в активной поездке.
5. Статус машины меняется на `away`.
6. Создаётся запись в журнале (`departure_time = NOW()`, `driver_id`).

### 3.3. Въезд автомобиля (админ)
1. Машина существует и находится в поездке (`status = 'away'`).
2. На парковке есть свободное место (`SELECT COUNT(*) FROM cars WHERE status = 'parked'` < 100).
3. Находится последняя незакрытая запись для этой машины (`return_time IS NULL`).
4. Запись закрывается (`return_time = NOW()`).
5. Статус машины меняется на `parked`.

### 3.4. Добавление новой машины (админ)
- Если мест на парковке **< 100**, статус `parked`.
- Если мест **≥ 100**, статус `away` (машина сразу уезжает).

### 3.5. Удаление машины (админ)
- Запрещено, если машина находится в поездке (`status = 'away'`).

### 3.6. Принудительное закрытие поездки (админ)
- Закрывает запись по `id`, статус машины → `parked`.
- Используется при сбоях (забыли отметить въезд).

## 4. Роли и доступ

| Действие | Жилец | Администратор |
|----------|-------|---------------|

| Просмотр своих поездок | ✅ | ✅ |

| Просмотр всех машин / жильцов / поездок | ❌ | ✅ |
| Выезд / въезд | ❌ | ✅ |
| Добавление / изменение / удаление машин | ❌ | ✅ |
| Добавление / изменение / удаление жильцов | ❌ | ✅ |
| Принудительное закрытие поездок | ❌ | ✅ |

## 5. API (эндпоинты)

### 5.1. Авторизация
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/auth/resident` | Вход жильца по номеру телефона → `resident_id` |
| POST | `/auth/admin` | Вход админа (логин `admin`, пароль `admin`) |

### 5.2. Жилец (требуется `resident_id` в сессии)
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| GET | `/resident/{id}` | Панель жильца (свои машины, поездки, статус парковки) |

### 5.3. Администратор (требуется роль admin в сессии)
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| GET | `/admin-dashboard` | Панель админа (таблицы машин и жильцов) |
| POST | `/exit` | Выезд |
| POST | `/entry` | Въезд |
| POST | `/cars` | Добавить машину |
| PUT | `/cars/{id}` | Изменить машину |
| DELETE | `/cars/{id}` | Удалить машину |
| POST | `/residents` | Добавить жильца |
| PUT | `/residents/{id}` | Изменить жильца |
| DELETE | `/residents/{id}` | Удалить жильца (проверка: нет машин) |
| GET | `/residents` | Список жильцов |
| POST | `/admin/close-trip/{trip_id}` | Принудительно закрыть поездку |
| GET | `/admin/open-trips` | Все открытые поездки |
| GET | `/admin/all-cars` | Все машины |
| GET | `/admin/all-logs` | Весь журнал |

## 6. Веб-интерфейс (Jinja2)

| Страница | Что содержит |
|----------|--------------|
| `/` | Главная: поле ввода телефона (жилец) + кнопка «Я админ» |
| `/admin-login` | Логин и пароль администратора |
| `/admin-dashboard` | Таблицы машин и жильцов, кнопки действий, свободные места |
| `/resident/{id}` | Свои машины, свои поездки, свободные места |

## 8. Развёртывание

- `start.bat` — запуск контейнеров в фоне, открытие браузера
- `stop.bat` — остановка контейнеров
- Всё работает через Docker Compose, без ручной установки зависимостей
